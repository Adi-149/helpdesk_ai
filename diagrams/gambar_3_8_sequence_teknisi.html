Gambar 3.8 Sequence Diagram Teknisi

sequenceDiagram actor T as Teknisi / Support participant B as Browser / View participant AC as AuthController participant TC as TicketController participant DB_U as Database (users) participant DB_T as Database (tickets) participant DB_H as Database (ticket_histories) Note over T,DB_H: Proses Login Teknisi T->>B: Mengakses halaman login B->>AC: POST /login (email, password) AC->>DB_U: SELECT * FROM users WHERE email=? DB_U-->>AC: Data user (role='support') AC-->>B: Redirect ke Dashboard Support B-->>T: Menampilkan Dashboard Support (statistik tiket) Note over T,DB_H: Melihat Daftar Tiket T->>B: Mengakses halaman Daftar Tiket B->>TC: GET /support/tickets TC->>DB_T: SELECT * FROM tickets ORDER BY created_at DESC DB_T-->>TC: Daftar seluruh tiket TC-->>B: Return view daftar tiket B-->>T: Menampilkan daftar tiket Note over T,DB_H: Melihat Detail dan Self-Assign Tiket T->>B: Memilih tiket untuk dilihat detail B->>TC: GET /tickets/{id} TC->>DB_T: SELECT * FROM tickets WHERE id=? (termasuk kolom AI) DB_T-->>TC: Data tiket + ai_summary, ai_causes, ai_recommendations TC-->>B: Return view detail tiket B-->>T: Menampilkan detail tiket + analisis AI + kasus serupa T->>B: Klik tombol Self-Assign B->>TC: POST /tickets/{id}/assign (assigned_to=self) TC->>DB_T: UPDATE tickets SET assigned_to=teknisi_id, status='progress' DB_T-->>TC: Berhasil TC->>DB_H: INSERT INTO ticket_histories (old_status, new_status='progress') DB_H-->>TC: Riwayat tersimpan TC-->>B: Redirect kembali ke detail tiket B-->>T: Menampilkan tiket dengan status 'progress' Note over T,DB_H: Update Status & Prioritas Tiket T->>B: Mengubah status/prioritas tiket B->>TC: PUT /tickets/{id}/status (status, priority, notes) TC->>DB_T: UPDATE tickets SET status=?, priority=? DB_T-->>TC: Berhasil TC->>DB_H: INSERT INTO ticket_histories (old_status, new_status, notes) DB_H-->>TC: Riwayat perubahan tersimpan TC-->>B: Redirect dengan pesan sukses B-->>T: Menampilkan konfirmasi perubahan
